Endpoint /pet (створення тваринки)

Позитивні:

 1. Перевірити, що запит працює з усіма параметрами (обов'язковими + необов'язковими) 
 2. Перевірити, що запит працює лише з обов'язковими параметрами. 
 3. Перевірити, якщо в масиві photoUrls декілька значень, тоді запит працює успішно. 
 4. Перевірити, якщо в масиві tags декілька об'єктів, тоді запит працює успішно. 
 5. Перевірити, якщо запит відпрацював успішно, статус-код відповіді 200. 
 6.Перевірити, що формат відповіді JSON. 
 7. Перевірити, що значення параметрів у відповіді повністю співпадають із значеннями, які передані у Body. 
 
 Негативні:
 
 8. Перевірити, що без обов'язкового параметру (наприклад id), у відповіді повертається помилка 400 Bad Request.
 9. Перевірити, якщо в Body передається тип даних !int (наприклад string) для параметру id, тоді у відповіді повертається помилка 404 Not Found.   
10. Перевірити, якщо значення параметру status not in ('available','pending','sold') (наприклад send), тоді у відповіді повертається помилка 400 Bad Request.
